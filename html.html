HTML Skeleton // Boilerplate

!DOCTYPE html is a flag that tells the browser that you are using the latest version of HTML.

Elements:

  • html: is the root element in our document.
  • head: contains the metadata of our document.
  • tittle: defines the title of our webpage. It is displayed in the tab of the browser and is used in search engines.
  • body: represents all of the content of our webpage.
  • ol: ordered list
    1. Step 1
    2. Step 2
    3. Step 3
  • ul: unordered list
    • Milk
    • Egg
    • Cheese
  • li: list item - is used to represent a single item in a list
  • a tag: the anchor element is used to create links to anything a URL can address. Anchor tags use the href (hypertext reference) attribute to link the destination. Google.com. The a tag is an inline element and will not be created on a new block.
  • The image tag is a self closing tag that is used to insert an image into your webpage. It uses the src attribute to point to the location of your image.

HTML5

HTML5 is the latest evolution of standard that defines HTML. The HTML standard is a document that describes how HTML should work. The standard describes the rules of HTML, but browsers actually have to do the work and implement HTML according to those rules. HTML5 is the latest evolution of the standard that defines HTML. It includes new elements and features for browsers to implement.

Inline and Block Elements

  • Inline elements fit in alongside other elements.
  • Block level elements take up a whole "block" of space.

Div

Div is a generic block level container to group elements/content together.

Span

Span is a generic inline container.

Odd Elements

  • hr: a self closing thematic break element. Default styling will be a line that goes across the screen.
  • br: produces a line break in text.
  • sup: is used to designate text that is superscript. It is text that is elevated off of the baseline. Similar to an exponent. 12
  • sub: will create text that is below the baseline. 12

Entity Code

Start with an ampersand and ends with a semicolon. It is used to display reserved characters. ♥

Semantic Markup

Semantic - relating to meaning.

  • main - represents the dominant content of your page.
  • header - any sort of introductory content
  • section - represents a standalone section that doesn't have a more specific semantic element.
  • nav - represents a section of a page that contains navigational links.
  • footer - represents the footer of a document.
  • article - represents a self-contained composition in a document page, application, or site.
  • aside- represents a portion of your document that is indirectly related to the main content.

Semantic markup can help with SEO and more crawlable to other applications. It also helps with navigation for screen readers.

Tables

Tables are structured sets of data made up of columns and rows.

  • table: represents tabular data.
  • td: defines a cell of a table that contains data.
  • tr: represents a row of cells in a table.
  • th: defines a header in a table.
  • thead, tbody, tfoot are other elements that are used in tables.
  • The rowspan attribute specifies the number of rows a cell should span.
  • The colspan attribute defines the number of columns a cell should span.

Forms

The form element itself is a shell or container that doesn't have any visual impact. The form element dictates where the form data should be sent. The action attribute is used for the destination. When you submit a form, a HTTP request will be made. The method attribute specifies which HTTP method should be used.

  • input: The input element is used to create a variety of different form controls.
  • label: associates corresponding text to the input. Great for assistive technology.
  • button: represents a clickable button.
  • The name attribute in a form is used as a reference when the data is submitted.
  • checkbox, radio(value), select, range, textarea
